Statistick prce (Vliv CT opening-killu na pravdpodobnost vhry kola v Counter-Strike 2)

Pravdpodobnost a statistika 1 (NMAI059) - LS 2024/25

vod

Counter-Strike 2 je tmov taktick stleka, v n se proti sob utkvaj terorist (T side) a protiteroristick jednotka (CT side) v krtkch kolech na rznch mapch. Hri se sna splnit specifick cle (poloen nebo znekodnn bomby) i eliminovat soupee, piem zsadn roli hraje spoluprce, pesnost a strategie.

V tto prci se zamuji na vznam tzv. opening killu prvn eliminace v kole a jeho vliv na vsledek kola. Konkrtn zkoumm, zda zskn opening killu protiteroristickou stranou skuten zvyuje pravdpodobnost jejho vtzstv. CT strana je v centru analzy proto, e na zatku kola se typicky rozmst do obrannch pozic, zatmco T strana sousted sv sil na prnik k jedinmu cli. V dsledku toho bv opening kill ze strany T obvykle silnm prediktorem vhry, zatmco vliv opening killu ze strany CT nen zcela jednoznan a stoj za bli zkoumn.

Cle

  • Zjistit, jak siln opening kill (dle v textu jako OK) souvis s vhrou kola.
  • Porovnat efekt po mapch.
  • Ovit, zda loadout (AWP = typ zbran) souvis s:
    • (a) anc zskat OK
    • (b) conversion tj. promna OK ve vhru kola.
  • Bonus: krtk asov analzy (OK timing, trade latence).

Trade ve videohrch oznauje situaci, kdy hr zabije soupee, ale je krtce pot eliminovn jinm protihrem, take oba tmy pijdou o jednoho hre.

from scipy.stats import chi2_contingency, fisher_exact
import matplotlib.pyplot as plt

with sqlite3.connect(DB_PATH) as con:
    kills = pd.read_sql("""
        SELECT match_id, demo_file, round, victimSteamID, headshot
        FROM kills_slim
    """, con)
    load = pd.read_sql("""
        SELECT match_id, demo_file, round, steamid, has_helmet
        FROM round_loadouts
    """, con)

# Piprav flagy
kills["headshot"]   = pd.to_numeric(kills["headshot"], errors="coerce").fillna(0).astype(int)
load["has_helmet"]  = pd.to_numeric(load["has_helmet"], errors="coerce").fillna(0).astype(int)

# Join: k obti v danm kole piadme stav helmy (snapshot po freeze)
D = kills.merge(
    load.rename(columns={"steamid":"victimSteamID"}),
    on=["match_id","demo_file","round","victimSteamID"],
    how="left"
).dropna(subset=["has_helmet"])

# 22 kontingenn tabulka v poad index/columns = [0,1]
tab = pd.crosstab(D["has_helmet"], D["headshot"]).reindex(index=[0,1], columns=[0,1], fill_value=0)

# --- Testy
chi2, p_chi2, dof, exp = chi2_contingency(tab.values)
or_fisher, p_fisher = fisher_exact(tab.values)  # matice [[noHelmet noHS, noHelmet HS],[Helmet noHS, Helmet HS]]

# --- Mry, Wilsonovy 95% CI
def wilson_ci(k, n, z=1.96):
    k, n = float(k), float(n)
    if n <= 0: return (np.nan, np.nan, np.nan)
    p = k/n
    denom = 1 + z**2/n
    center = (p + z**2/(2*n))/denom
    half   = z*np.sqrt((p*(1-p)/n) + (z**2/(4*n**2))) / denom
    return (p, center - half, center + half)

# Poty pro kadou skupinu
n0 = tab.loc[0].sum(); hs0 = tab.loc[0,1]; nohs0 = tab.loc[0,0]
n1 = tab.loc[1].sum(); hs1 = tab.loc[1,1]; nohs1 = tab.loc[1,0]

p0, lo0, hi0 = wilson_ci(hs0, n0)
p1, lo1, hi1 = wilson_ci(hs1, n1)

# OR (stejn uspodn jako u fisher_exact ve): [[b, a],[d, c]]
b, a = nohs0, hs0
d, c = nohs1, hs1
OR = (b*c) / max(1, a*d)
se_logOR = np.sqrt(1/max(1,b) + 1/max(1,a) + 1/max(1,d) + 1/max(1,c))
OR_lo, OR_hi = np.exp(np.log(OR) - 1.96*se_logOR), np.exp(np.log(OR) + 1.96*se_logOR)

# --- Hezk tabulky

ct.index   = ["Bez helmy (0)", "S helmou (1)"]

ct.loc[" sloupec"] = ct.sum()

import sqlite3, pandas as pd, numpy as np
from scipy.stats import mannwhitneyu
import matplotlib.pyplot as plt

DB_PATH = "data/warehouse.sqlite"  #  zm podle sebe

# ---------- Naten a pprava dat ----------
with sqlite3.connect(DB_PATH) as con:
    header = pd.read_sql("SELECT match_id, demo_file, tickrate FROM header", con)
    rounds = pd.read_sql("SELECT match_id, demo_file, round, start, end FROM rounds", con)
    bomb   = pd.read_sql("""
        SELECT match_id, demo_file, round, LOWER(event) AS event
        FROM events_bomb
    """, con)

# Flag plantu v kole
plants = (bomb.query("event == 'planted'")
              .drop_duplicates(["match_id","demo_file","round"])
              .assign(planted=1)[["match_id","demo_file","round","planted"]])

R = (rounds.merge(header, on=["match_id","demo_file"], how="left")
            .merge(plants, on=["match_id","demo_file","round"], how="left")
            .fillna({"planted":0})
            .dropna(subset=["start","end","tickrate"]))

# Dlka v sekundch
R = R.assign(dur_s=(R["end"] - R["start"]) / R["tickrate"])

with_plant = R.loc[R["planted"]==1, "dur_s"].to_numpy()
no_plant   = R.loc[R["planted"]==0, "dur_s"].to_numpy()

n1, n0 = len(with_plant), len(no_plant)
if n1 == 0 or n0 == 0:
    raise ValueError("Chyb data v jedn z vtv (plant/no-plant).")

# ---------- Souhrnn tabulka ----------
def five_num(x):
    x = np.asarray(x)
    return pd.Series({
        "n": len(x),
        "mean_s": np.mean(x),
        "std_s": np.std(x, ddof=1) if len(x) > 1 else np.nan,
        "q1_s": np.percentile(x, 25),
        "median_s": np.median(x),
        "q3_s": np.percentile(x, 75),
    })

summary = pd.concat([
    five_num(no_plant).rename("No plant (0)"),
    five_num(with_plant).rename("Plant (1)")
], axis=1).T

# rozdl medin (plant  no-plant)
median_diff = np.median(with_plant) - np.median(no_plant)

# ---------- Test + efektov mra ----------
# MannWhitney U: p-value z dvoustrann varianty
u_two, p_two = mannwhitneyu(with_plant, no_plant, alternative="two-sided")

# Pro efektov mry potebujeme U pro hypotzu "plant > no-plant"
u_greater, p_greater = mannwhitneyu(with_plant, no_plant, alternative="greater")
N = n1 * n0

# Common Language Effect Size = P(X>Y) + 0.5 P(X=Y) ~ U/N
cles = u_greater / N

# Rank-biserial correlation (RBC) = 2*U/N - 1  (pro smr "plant del ne no-plant")
rbc = 2*(u_greater/N) - 1

# ---------- Hezk tisk tabulek ----------
pd.options.display.float_format = '{:,.3f}'.format

print(summary.to_string())

from scipy.stats import mannwhitneyu
import matplotlib.pyplot as plt

DB_PATH = "data/warehouse.sqlite"

with sqlite3.connect(DB_PATH) as con:
    header = pd.read_sql("SELECT match_id, demo_file, map_name, tickrate FROM header", con)

rows = []
for m, g in R.groupby("map_name"):
    a = g.loc[g.planted==1, "dur_s"].to_numpy()

    if len(a)==0 or len(b)==0: continue

                 "med_plant_s": np.median(a), "med_no_s": np.median(b),
                 "diff_med_s": diff, "p_mwu": p})

         .sort_values("diff_med_s", ascending=False)
         .reset_index(drop=True))

def or_ci(a, b, c, d):
    # Odds ratio a 95% CI (Woolf; s Haldane-Anscombe 0.5 pi nulch).
    a,b,c,d = map(float, (a,b,c,d))
    if min(a,b,c,d) == 0:
        a += 0.5; b += 0.5; c += 0.5; d += 0.5
    OR = (a*d)/(b*c)
    se = math.sqrt(1/a + 1/b + 1/c + 1/d)
    lo = math.exp(math.log(OR) - 1.96*se)
    hi = math.exp(math.log(OR) + 1.96*se)
    return OR, lo, hi

def trade_curve_fast(ks: pd.DataFrame, hdr: pd.DataFrame, T_max=12) -> pd.DataFrame:
    if ks.empty:
        return pd.DataFrame(columns=["T","ret_rate"])

    df = ks.copy()
    df["killerSide"] = df["killerSide"].str.lower()

    # prvn kill v kole
    fk = (df.sort_values("tick")
            .groupby(["match_id","demo_file","round"], as_index=False)
            .first()
            .rename(columns={"tick":"t0","killerSide":"side0"}))

    # pipoj tickrate per demo (fallback na default)
    fk = fk.merge(hdr, on=["match_id","demo_file"], how="left")
    fk["tickrate"] = pd.to_numeric(fk["tickrate"], errors="coerce").fillna(tickrate_default).astype(float)

    # vechny pozdj killy opan strany v tomt kole
    jj = (df.merge(fk, on=["match_id","demo_file","round"], how="inner"))
    jj = jj[(jj["tick"] > jj["t0"]) & (jj["killerSide"] != jj["side0"])]

    # minimln delta asu do prvnho proti-killu v kole
    if jj.empty:
        n_rounds = len(fk)
        return pd.DataFrame({"T": np.arange(1, T_max+1), "ret_rate": np.zeros(T_max)})

    jj["dt_sec"] = (jj["tick"] - jj["t0"]) / jj["tickrate"]
    dt_min = (jj.groupby(["match_id","demo_file","round"], as_index=False)["dt_sec"]
                .min()
                .rename(columns={"dt_sec":"dt_min_sec"}))

    # pipoj na vechny kola (kde trade nen, nechme NaN)
    fk2 = fk.merge(dt_min, on=["match_id","demo_file","round"], how="left")

    # vektorizovan spotat ret_rate(T): P(dt_min <= T)
    n_rounds = len(fk2)
    vals = fk2["dt_min_sec"].to_numpy()
    vals = vals[np.isfinite(vals)]  # jen kola, kter mla trade (ostatn > T vdy False)
    vals.sort()

    Ts = np.arange(1, T_max+1, dtype=float)
    # pro kad T kolik dt_min <= T (binary search)
    idx = np.searchsorted(vals, Ts, side="right")
    ret_rate = idx / float(n_rounds)
    return pd.DataFrame({"T": Ts, "ret_rate": ret_rate})

  • Opening kill vhra kola (celkem): velmi siln efekt. 22 tabulka: winneropening_side OR = 4.95 (95% CI 4.355.64), p 2.410, Fisher p 4.310. Strana s opening killem vyhrla 69 % kol.

Odpov na hlavn otzku

  • Ano. Opening kill je nejvt singulrn faktor spchu kola, konzistentn nap mapami (sla efektu se li).
  • Vsledek ale moduluje kontext:
    • Vbava (AWP, celkov sla loadoutu) opening efekt zesiluje.
    • Trade okno je krtk zen rizika v prvnch 35 s po OK je klov.
    • Stranov bias celkov vyrovnan, mapov vak existuj vznamn odchylky.

Praktick dopady (pro interpretaci/strategii)

  • Po zskn OK okamit snit trade-risk (ustoupit, pehrt hel, utility) nejdsivj je okno do 5 s.
  • AWP m mitelnou hodnotu v zisku OK i konverzi obzvl v kolech s rovnou/lehce slab zbytkovou vbavou.
  • Helmy se vyplcej: markantn snen pravdpodobnosti smrti headshotem.
  • Pro underdoga ani OK sm o sob nesta; je nutn rychl weapon pickup/upgrade nebo okamit tradovateln postup.

Limity a hrozby validity

  • Loadout snapshot je po freeze; nezachyt 100 % drop/pickup dynamiky bhem kola.
  • Kategorizace vbavy je token-based (bez cen/equip value); zvolen ven pro score jsou heuristika.
  • Per-map testy mimo vslovn korigovan tabulky nejsou korigovan na vcensobn testovn.
  • Nezohledujeme skill/elo tm, server/turnajov kontext a pesn role hr.

Mon pokraovn

  • Vzt ekonomickou hodnotu (equip value) a modelovat konverzi OK s kvantitativnmi delty (msto heuristickho score).
  • Per-map/per-site modely (A/B na dan map) a as-do-plant jako kovarita.
  • Causal pstupy (matching / IPTW) pro robustnj odhad dopadu AWP/helm/SMG apod.
